# B2B Portal — Демонстрация функционала (Апрель–Май 2026)

## 1. СИСТЕМА ПОЛЬЗОВАТЕЛЕЙ ✅

### Регистрация и вход
| Сценарий | Как проверить |
|----------|--------------|
| Вход ADMIN | `admin@test.com` / `admin` — полный доступ |
| Вход MANAGER | `manager@test.com` / `manager` — без 2FA |
| Вход USER | Создать через админа → вход с SMS-2FA |
| Защита от брутфорса | 5 неверных паролей → блокировка 15 мин |
| Inactivity timeout | USER 30 мин бездействия → разлогин |
| Session superseded | Войти с другого устройства → первая сессия завершена |
| Сброс пароля | `/forgot-password` → письмо на email |
| Смена пароля | Принудительная смена при первом входе |

### 2FA
- 6-значный код по SMS (Smart Sender A1)
- 3 неверных кода → блокировка 2FA на 15 мин
- 60 сек между повторными отправками
- Код живёт 5 минут

## 2. АДМИН-ПАНЕЛЬ ✅

### Мониторинг (вкладка "Мониторинг")
- Статистика: всего пользователей, онлайн
- При наведении на "Пользователи: N" — тултип с именами кто онлайн

### Список пользователей (вкладка "Список пользователей")
- Поиск по названию компании/email/имени
- Пагинация
- Блокировка / разблокировка
- Удаление пользователя
- Видно кто в онлайне (зелёная точка)
- Индикаторы блокировки: парольная, 2FA, ручная

### Создание пользователя (вкладка "Создать пользователя")
- Выбор компании из списка (УНП подтягивается)
- Роли: USER / MANAGER
- После создания — welcome email с паролем

### Логи (вкладка "Логи")
- Просмотр логов сервера
- Фильтр по уровню (info/warn/error)
- Поиск по тексту
- Выбор даты

### Настройки (вкладка "Настройки")
- CRUD настроек портала

### Table Browser (вкладка "Просмотр и редактирование БД")
- Просмотр таблиц (Company, User, Project, Message, SiteSetting)
- Защита: whitelist таблиц, readOnly поля
- Редактирование строк
- Пагинация

### Бэкапы (вкладка "Восстановление и бэкап БД")
- Создание бэкапа (pg_dump)
- Список бэкапов
- Скачивание
- Удаление
- Восстановление из бэкапа
- Загрузка .sql файла
- Cron-расписание (пресеты + своё выражение)

## 5. БЕЗОПАСНОСТЬ ✅

- JWT access (15m) + refresh token rotation (7d)
- Refresh token: SHA-256 хеш в БД, reuse detection
- httpOnly cookies, SameSite Strict
- Rate limiting: 500 req/15min общий, 3 req/min 2FA send
- Helmet CSP
- Brute force защита: вход (5 попыток) + 2FA (3 попытки)
- ReadOnly поля в DB Browser

## Сценарий полной демонстрации

1. **Открыть** `http://localhost:5001/api-docs` — показать Swagger
2. **Войти как ADMIN** → показать мониторинг, пользователей, логи, Table Browser, бэкапы
3. **Создать USER** → показать форму регистрации с выбором компании
4. **Выйти, войти как USER** → показать 2FA по SMS (код в консоли сервера)
5. **Показать обзор USER** → новости, заглушки 1С
6. **Войти как MANAGER** → показать проекты, оборудование, рассылки, новости, историю
7. **Запустить тесты** → `cd server && npm test && cd ../client && npm test` — показать 76 зелёных
